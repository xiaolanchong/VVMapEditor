работа преобразований.

	* Сразу может быть включено несколько преобразований. 
	* Преобразования выстроены в очередь в следующей последовательности
		(1) Триангуляция.
		(2) Нейронное.
		(3) Проективное.
		последовательность может быть легко изменена, но требуется пересборка проекта
	* работа GetXY:
		преобразования вызываются по очереди начиная с начала.
 		если какое-то преобразование сработало успешно, считаем полученный результат правильным.
		неуспех может быть в силу следующих причин: 
			* оно выключено (снят флажок Enabled)
			* оно не откомпиллированно, либо обнаружено, что исходные данные (точки) изменились
			* исходная точка вне зоны интерполяции (для Триангуляция и Нейронное)
			* преобразование не может быть выполнено для данной точки (горизонт для проективного)
	* компиляция:
		компиляция выполняется только для включенных (Enable -> true) преобразований
		компиляция выполняется с конца очереди

	* при загрузке старого формата включено только проективное		

пояснения новым преобразованиям:

	нейронное
	
	* работает только в области интерполяции

	* в настройках: 
		формула нейронной сети - набор целых чисел, разделяемых пробелами
		число итераций - число итераций, выполняемое при компиляции.
			внимание! если задать слишком большое число, расчет может затянуться

	триангуляция

	* работает только в области интерполяции

	* если сработало, качество преобразования должно равняться 100, т.к. все установленные точки идеально переводятся в себя

show transition log window 
	
	если включено, то при вызове GetXY появится TOP_MOST (которое поверх всех окон) окно, куда будет отображаться результат работы GetXY

ЗЫ 
	область интерполяции - область где могут выполняться алгоритмы интерполяции. Это область, ограниченная минимальным полигоном, 
	включающего все точки учавствующие в конкретном преобразовании.
